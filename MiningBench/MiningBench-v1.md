面向大数据的多步骤推理的数据挖掘benchmark：规则因子挖掘

1、背景原理
基于密码学的“验证不对称”原则设计Benchmark：生成规则容易，逆向破解极难。


2、目标
测试模型通过多轮工具调用和多轮推理来完成数据挖掘任务的端到端能力。
数据挖掘是指从大量数据中挖掘出有价值的模式、规律、关系或异常情况的一系列工作。
本文聚焦的是数据挖掘中最经典的一类任务（规则挖掘/因子挖掘），具体而言是指，在涵盖众多变量的数据中，围绕着其中一个目标变量，通过数学推理、代码执行、机器学习等手段，找到与目标变量最相关的变量，以及影响目标变量的具体规则。
为了让任务尽量贴近现实世界，我们从金融行业的风控因子/风控规则、量化交易行业的量化因子/规则中获取灵感，提取其中的主流规则（如 求比值、求和等），作为预定义的规则模式。

3、数据挖掘（规则因子挖掘）的任务特点 / 研究亮点
（1）需要多步骤推理: 可能
（2）需要工具调用：基于python代码实现的各种数据分析方法
（3）问题无通解：在实际的工业场景中，也高于依赖于数据挖掘专家的非标准化的、启发式的探索/推理/试错
（4）无法被预先计划（unplanned）：任务难度无法在开始阶段预估，需要依赖于后续的探索结果、环境反馈来逐步了解到任务的实际难度
（5）高度开放的动作空间：与多跳搜索类动作不同 ，数据挖掘类任务的解决可能各种启发式的探索动作（相关系数矩阵、分布对比、树模型、特征重要性分析、SHAP值），具有极高的难度。
（6）问题难度大：对于非线性、组合式的规则，通常需要极大的推理难度。
（7）噪声：真实的数据挖掘任务中包含着复杂的噪声，导致无法像数学问题推理一样，在推理的中间阶段，获得的线索信号更加稀疏和不置信。

4、Benchmark的半自动化构建方案
（1）生成初始数据
生成200份随机数据，每份数据包含N+1个方差、均值不同的随机变量，包含N个潜在的因变量和1个目标变量。
（2）生成构造规则作为groudtruth
在预定义的规则形式中随机选择 1 种作为 groundtruth。这些规则模式均来自金融风控和量化交易中常见的因子构造方式，并按照“类别编号 + 正式名称 + 数学表达式 + 业务解释”的格式统一描述。
说明：
    - 记 `X1, X2, X3` 为候选特征，`β1, β2, β3` 为待估系数，`N` 为判定阈值，`L/U` 为区间下界与上界，`K` 为门槛参数。部分单调非线性变换规则采用连续回归目标，以避免其在二分类阈值任务中退化为等价的单变量线性阈值。
    - 对于涉及 `log`、开方、比值的规则，实际生成数据时应保证定义域合法，例如保证变量为正、分母非零。
构建以下20类规则
    线性加权类
        1. 单变量线性阈值规则（难度：easy）  
        `Target = 1 if X1 > N else 0` 或 `Target = 1 if X1 < N else 0`  
        风控：收入高于阈值、逾期天数高于阈值；量化：单一动量因子超过阈值。
        2. 双变量线性组合阈值规则（难度：easy）  
        `Target = 1 if (β1 * X1 + β2 * X2) > N else 0`  
        风控：收入与负债联合决定风险；量化：价值因子与动量因子线性合成。
        3. 三变量线性组合阈值规则（难度：medium）  
        `Target = 1 if (β1 * X1 + β2 * X2 + β3 * X3) > N else 0`  
        多个弱因子线性叠加形成总评分。
    单变量非线性变换类
        4. 单变量三次放大规则（难度：medium）  
        `Target = X1**3 + C + noise`  
        回归任务。风控：异常交易金额的高阶放大；采用连续目标以保留三次曲线形状，而不是退化为 `X1 > N_prime`。
        5. 单变量平方放大规则（难度：easy）  
        `Target = 1 if X1**2 > N else 0` 或 `Target = 1 if X1**2 < N else 0`  
        平方会放大极端值，适合表达“越极端越危险”的非线性效应。
        6. 单变量平方根压缩规则（难度：medium）  
        `Target = X1**0.5 + C + noise`
        回归任务。开方用于表达边际影响递减；采用连续目标以保留压缩曲线形状。
        7. 单变量绝对偏离规则（难度：medium）  
        `Target = 1 if abs(X1) > N else 0` 或 `Target = 1 if abs(X1) < N else 0`  
        有些场景不关心方向，只关心“偏离程度”或“波动程度”。
        8. 单变量对数压缩规则（难度：medium）  
        `Target = log(X1) + C + noise`
        回归任务。对数用于压缩极端值；采用连续目标以保留对数曲线形状。
    相对关系与比例差异类
        9. 双变量对数比值规则（难度：hard）  
        `Target = 1 if log(X1 / X2) > N else 0` 或 `Target = 1 if log(X1 / X2) < N else 0`  
        量化：`log(现价 / 过去价格)`。`log` 可以压缩极端值，也让比例关系更平滑，金融里很常见。
        10. 双变量差值规则（难度：easy）  
            `Target = 1 if X1 - X2 > N else 0` 或 `Target = 1 if X1 - X2 < N else 0`  
            风控：当前收入 - 最低必要支出。差值代表“边际空间”或“趋势强弱”。
        11. 双变量平方比值规则（难度：hard）  
            `Target = 1 if (X1 / X2)**2 > N else 0` 或 `Target = 1 if (X1 / X2)**2 < N else 0`  
            当分子相对分母异常偏大时，风险/收益信号被非线性放大。
    交互与组合效应类
        12. 双变量交互乘积规则（难度：medium）  
            `Target = 1 if X1 * X2 > N else 0` 或 `Target = 1 if X1 * X2 < N else 0`  
            风控：高金额 * 高频次共同放大风险；量化：价格变化 * 成交量变化形成确认信号。
            当 N=0，用于判断双变量同号一致性，例如财务造假的利润与现金流，正常两者同号（同增同减）。若一正一负可能存在造假嫌疑。
        13. 双变量线性与交互混合规则（难度：hard）  
            `Target = 1 if (β1 * X1 + β2 * X2 + X1 * X2) > N else 0`  
            具有调节效应的因子影响，即线性影响叠加交互影响。
        14. 二次交叉组合规则（难度：hard）  
            `Target = 1 if β1 * X1**2 + β2 * X2 > N else 0`  
            一个变量的极端值被放大，同时叠加另一个线性影响。
    阈值、区间与截断类
        15. 单变量中心偏离规则（难度：hard）  
            `Target = 1 if abs(X1 - K) > N else 0`  
            风控或量化中常见“偏离正常水平越远，风险或机会越强”的规则，例如交易金额、价格、利用率相对常态中心的异常偏离。
        16. 多变量最大值截断规则（难度：hard）  
            `Target = 1 if max(β1 * X1, β2 * X2, β3 * X3) > N else 0`  
            木桶效应：风控中可对应 `max(多头借贷平台数, 历史最高逾期天数, 法院被执行次数)`。
        17. 单变量双边区间规则（难度：medium）  
            `Target = 1 if L < X1 < U else 0`
        18. 双变量线性组合双边区间规则（难度：hard）  
            `Target = 1 if L < (β1 * X1 + β2 * X2) < U else 0`  
            量化：换手率落在某个中间区间时，未来收益更差。
    条件门控类
        19. 双变量AND门控（难度：hard）  
            `Target = 1 if (X2 > K) and (X1 > N) else 0` 或 `Target = 1 if (X2 > K) and (X1 < N) else 0`  
            风控：只有当“多头借贷平台数 > 3”时，“近 30 天申请次数”才显著增加违约风险。
        20.  双变量OR门控（难度：hard）  
            `Target = 1 if (X1 > K) or (X2 > N) else 0` 或 `Target = 1 if (X1 > K) or (X2 < N) else 0`
            风控：任一风险信号触发即判定高风险，例如"历史曾有严重逾期（X1 > K）"或"近30天申请次数过多（X2 > N）"，满足其一即触发预警。
（3）将groudtruth的影响注入初始数据
将20个规则每个生成10个case，按规则将影响注入对应的200份数据，对应200个task
（4）添加噪声（可选）
对处理后的数据添加噪声
（5）semantic 版本任务合成
目前已生成的200个task作为”数据匿名任务“，所有变量的变量名都是”x1、x2“这种无实际意义的字段；
调用llm对每个”数据匿名任务“，基于groudtruth，生成每个字段的变量名（英文），groudtruth对应的变量及公式应具有一定的可解释性，其余变量也具备类似的业务含义，作为另外的200个 semantic 任务（数据可见任务）

5、数据规模
总体规模：
    - 每条规则生成 `10` 个 case，共 `20` 条规则；
    - 每个 case 生成两种任务版本：`anonymous`（匿名变量）与 `semantic`（可解释变量名）；
    - 总任务数：`10 * 20 * 2 = 400`。
按任务类型拆分：
    - `anonymous`：`200`（20 规则 * 10 case）
    - `semantic`：`200`（20 规则 * 10 case）
按目标类型拆分：
    - 二分类规则：17 条规则（除 4,6,8 外）-> 匿名 `170`，可见 `170`，合计 `340`
    - 回归规则：3 条规则（4,6,8）-> 匿名 `30`，可见 `30`，合计 `60`
按难度拆分（基于第 4 节难度标注）：
    - `easy`：4 条规则（1,2,5,10）-> 匿名 `40`，可见 `40`，合计 `80`
    - `medium`：7 条规则（3,4,6,7,8,12,17）-> 匿名 `70`，可见 `70`，合计 `140`
    - `hard`：9 条规则（9,11,13,14,15,16,18,19,20）-> 匿名 `90`，可见 `90`，合计 `180`

按规则大类拆分：
    - 线性加权类（3 条）-> 匿名 `30`，可见 `30`，合计 `60`
    - 单变量非线性变换类（5 条）-> 匿名 `50`，可见 `50`，合计 `100`
    - 相对关系与比例差异类（3 条）-> 匿名 `30`，可见 `30`，合计 `60`
    - 交互与组合效应类（3 条）-> 匿名 `30`，可见 `30`，合计 `60`
    - 阈值、区间与截断类（4 条）-> 匿名 `40`，可见 `40`，合计 `80`
    - 条件门控类（2 条）-> 匿名 `20`，可见 `20`，合计 `40`

（2）答案的输出格式：

（3）评估指标：
规则评估:
- 结果正确：Exact Match（严格），输出正确的规则(变量正确、形式正确、系数正确)，二元判断，考虑 Symbolic Equivalence（代数等价）的情况
- 回归任务补充：可在 held-out 数据上用 `RMSE/MAE/R2` 验证连续表达式的数值拟合质量，并结合表达式复杂度约束，避免用高阶近似或黑盒模型替代简洁生成式。
- 变量正确：规则匹配(EM)，判断是否找出了全部变量因子
- 形式正确：llm-as-judge，判断答案的规则形式是否正确
rubric评估（llm-as-judge，使用 0/1/2 三档评分）:
- 探索策略有效性：评估探索是否有针对性、结构化，而非盲目堆砌步骤。0分：直接猜测，无数据验证，或仅做单一分析后立即给出答案；1分：做了2步以上验证，但方法单一（如全部是相关系数），未覆盖非线性或组合特征；2分：系统性覆盖候选变量筛选 → 候选规则形式识别（含非线性/组合判断）→ 参数估计（系数/阈值）完整闭环。
- 假设驱动与证伪能力：评估模型是否以可证伪假设为基础进行推理，而非随机游走。0分：无明确假设，行为随机，或仅有一次猜测；1分：能提出假设并验证，但被证伪后直接换新假设，未分析失败原因；2分：形成"提出假设 → 设计检验 → 分析失败原因（如'相关系数低说明是非线性关系'）→ 有针对性修正"的完整证伪闭环。
- 推理-答案一致性：评估最终答案是否与模型自身的中间分析结论一致。0分：答案与中间分析明显矛盾（如中间已排除某变量但最终答案仍包含，或中间验证提示非线性但输出线性规则）；1分：答案与主要中间结论一致，但忽略部分细节（如系数量级明显偏差）；2分：答案与全部中间推理结论自洽，逻辑链完整可追溯。

7、JSONL 格式规划
将每个 task 存为 JSONL 的 1 行（一个样本一行），结构如下：

格式示例：
```json
{
  "id": "nb_v1_task_0001",
  "benchmark": "nice_bench_rule_mining",
  "split": "anonymous",
  "difficulty": "medium",
  "input_data_path": "mock_v1/task_01/data.csv",
  "allowed_tools": ["python"],
  "output_schema": {
    "answer_features_tag": "answer_features",
    "answer_rule_tag": "answer_rule"
  },
  "ground_truth": {
    "rule_name": "双变量线性与交互混合规则",
    "task_type": "classification",
    "answer_features": ["feat_1", "feat_4"],
    "answer_rule": "target = 1 if (2*feat_1 + -3*feat_4 + feat_1*feat_4) > 7 else 0",
    "parameters": {
      "noise_ratio": 0.1
    }
  },
  "metadata": {
    "n_rows": 1000,
    "n_features": 10,
    "task_seed": 20260406,
    "generator_version": "v1.2.0"
  }
}
```

变量含义：
- `id`：任务唯一标识
- `benchmark`：基准名称
- `split`：`anonymous`（匿名变量）、`semantic`（可解释变量名）
- `difficulty`：任务难度分层, `easy`、`medium`、`hard`。
- `input_data_path`：任务数据文件路径（csv格式）。
- `allowed_tools`：允许模型使用的工具白名单，这里固定为 `python`
- `output_schema`：输出标签规范，便于按规则解析答案
- `ground_truth`：标准答案与规则参数。
- `ground_truth.rule_name`：规则名称，对应 20 类规则。
- `ground_truth.task_type`：目标类型，`classification` 表示二分类规则，`regression` 表示连续回归规则。
- `ground_truth.answer_features`：规则真实使用到的全部变量，按表达式出现顺序。
- `ground_truth.answer_rule`：标准规则表达式，用于 EM 或结构匹配。
- `ground_truth.parameters`：仅保留噪声强度等评测必要参数，例如 `noise_ratio`。
- `metadata.n_rows / n_features`：数据规模信息
- `metadata.task_seed`：该任务随机种子，保证可复现
- `metadata.generator_version`：版本号

9、Prompt 敏感性分析
为评估提示词措辞对模型表现的影响，设计以下 prompt 变体进行消融实验：
（1）Hint 等级变体
在"任务提示"部分控制对规则形式的提示强度，分为三档：
- No-Hint（基线）：仅保留当前提示词，不给出任何关于规则形式的额外线索
- Weak-Hint：追加弱提示，如"规则可能涉及非线性变换或多变量组合"
- Strong-Hint：追加强提示，如"规则可能涉及对数、幂次、比值、交互项、截断或门控等非线性形式"
（2）分析维度
- 按 Hint 等级对比：No-Hint / Weak-Hint / Strong-Hint 下的 pass@1 和各过程指标
- 按难度交叉分析：Hint 对 easy/medium/hard 任务的提升幅度差异
- 按规则类别交叉分析：Hint 对不同规则大类（线性 vs 非线性 vs 门控）的影响是否一致
（3）目的
量化"提示词中的先验知识"对模型挖掘能力的影响程度，区分模型的"真实推理能力"与"依赖提示的猜测能力"。
